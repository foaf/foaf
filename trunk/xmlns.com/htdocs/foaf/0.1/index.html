See FOAF Wiki page for information on proposed additions to this namespace. Note that the FOAF namespace is a changing thing; we are not creating new namespace URIs with each addition and edit.

The general overview provides some background on the RDFWeb system. An RDFViz graphical representation of this namespace is also available. Note that the RDFWeb system can make use of a number of RDF vocabularies. The properties and types defined here provide some basic useful concepts for use in the RDFWeb. Other vocabulary (eg. the Dublin Core metadata elements for simple bibliographic description) can also be mixed in with FOAF terms.

FOAF is expressed as an RDF Schema, annotated with DAML to express the fact that a foaf:mbox uniquely picks out an individual.

FOAF:mbox is an UnambiguousProperty, in the DAML+OIL sense:

  • daml:UnambigousProperty element, which is a subclass of ObjectProperty. This asserts that an instance y can only be the value of P for a single instance x, i.e: there cannot be two distinct instances x1 and x2 such that both (x1,y) and (x2,y) are both instances of P. Notice that the inverse property of a UniqueProperty is always an UnambigousProperty and vice versa.
