RDFWeb and TAP Projects: FOAF, Wordnet, Web services etc.

This document provides a home page for TAP-based projects that relate to the RDFWeb project. More general information on TAP is available from the Stanford project site.

See the TAP list archives for more background. (not yet -- list in progress)

TAP/RDFWeb Projects

TAP provides a very useful, and pretty large, database of things that will likely be of interest to RDF applications on the Web. To date, RDFWeb apps have used Wordnet to exploit a large RDF taxonomy of noun terms. TAP adds to this by providing TAP identifiers for specific things, such as people, places, etc.

We expect to use TAP in RDFWeb stuff in a number of ways.

In the photo metadata and 'co-depiction' experiments, TAP is useful as it provides names for 100s of entities such as 'Frank Sinatra' and 'JFK' who we have previously found difficult to describe usefully in RDF. It provides a list of actors, which will help us describe movies. It provides some extensions to RDF Schema, which will help us evaluate the strengths and weaknesses of the evolving W3C WebOnt language. Initial ideas for projects include photo metadata, exploring a framework for integration / mapping with Wordnet (at least the noun terms we've mapped into RDF classes).

Online demo

Currently there is an experimental installation of TAP running here. It may not always be available, as it is on a development machine. It is currently the same as the main services running on the Stanford site.

Some files and links. The instance data in the current TAP db is dominated by music data. This is likely to change as TAP grows.

  • RDFWeb TAP installation
  • 300+ main types: taptypes.txt (a quick dump listing all used values of rdf:type / tap:instanceOf; all sensible hierarchy is flattened by the simple query)
  • ditto for tapproperties.txt -- or a bunch of them; need to write Perl to get them all
  • all tap types, not just the ones that are instantiated in the db (approx 6000)
